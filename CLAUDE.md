# 씨몬스터 내부도구 — 작업 규칙

이 저장소는 **여러 Claude Code 세션이 같은 폴더에서 동시에** 작업합니다.
워킹트리도 브랜치(`main`)도 **하나뿐**이라, 다른 세션의 편집이 같은 파일에 그대로 보입니다.
**git 은 이 상황에서 아무것도 막아주지 못합니다** — 합칠 게 없으니 충돌 마커도 없고, 나중에 저장한 쪽이 조용히 이깁니다.

`main` push = **즉시 프로덕션 자동배포** (Vercel). 브랜치 보호·PR 리뷰 없음. 되돌릴 수 없는 것이 나갈 수 있습니다.

---

## 1. 절대 쓰지 말 것

| 명령 | 결과 |
|---|---|
| `git add -A` / `git add .` | 다른 세션의 **미완성 코드가 프로덕션 배포됨** (광고 예산 API 등 되돌릴 수 없는 것 포함) |
| `git commit -am` | 새 파일만 누락 → main 빌드 실패 → **전 세션 배포 정지** |
| `git clean -fd` / `git stash -u` | 미추적 파일 **영구 소실** (reflog 로도 복구 불가) |
| `git checkout -- .` / `git reset --hard` | 다른 세션의 진행 중 작업 **영구 소실** |
| `npm ci` | dev 서버가 잡고 있는 네이티브 파일에서 멈춰 `node_modules` 반파 |
| `taskkill /PID ...` (dev 서버) | 다른 세션의 미리보기를 죽임. dev 에러 메시지가 이걸 시키지만 **따르지 말 것** |

## 2. 커밋은 반드시 경로를 명시

```bash
git status --short          # 내가 안 건드린 파일이 섞여 있는지 먼저 확인
git add app/sales/page.tsx app/api/sales/upload/route.ts   # 경로를 손으로 적는다
```

- `-A` / `.` / `-am` 금지. **새 파일도 이름을 직접 적는다.**
- `git status --short` 에 **내가 만들지 않은 변경이 보이면 그건 다른 세션의 작업이다.** 커밋하지 말고 그대로 둘 것.
- **한 몸인 파일은 반드시 같은 커밋에.** (화면만 커밋되고 서버가 빠지면 빌드는 통과하는데 기능이 조용히 죽는다.)

## 3. 동시에 하면 안 되는 작업 — 세션 수보다 이게 중요

세션 개수는 문제가 아니다. **아래 두 가지는 동시에 두 세션이 하면 안 된다.**

### (a) DB 구조 변경 (`supabase/migrations/`)

마이그레이션은 **사람이 Supabase SQL Editor 에 손으로 붙여넣어** 적용한다. CLI 도, 적용 이력 장부 테이블도 없다 —
**중복 번호를 잡아줄 주체가 파이프라인 전체에 하나도 없다.**

두 세션이 각각 `071_a.sql` / `071_b.sql` 을 만들면 파일명이 달라 **git 이 조용히 둘 다 통과시킨다**(재현 확인).
그리고 `create ... if not exists` 패턴 때문에 나중에 붙여넣은 쪽이 **에러 없이 무시되어**, 잘못된 번호가 실데이터에 박힌 뒤에야 발견된다.

> **DB 구조를 바꾸는 세션은 항상 하나만.** 채번 전 `ls supabase/migrations/ | tail -3` 로 확인하고, 만들었으면 즉시 공유.

### (b) 공용 파일 수정

내 기능만 고치는 것처럼 보이지만 전 서비스가 쓰는 파일:

| 파일 | 쓰는 곳 | 조용한 사고 |
|---|---|---|
| `app/lib/b2b-activity.ts` | 5개 영역 21개 파일 | `recordActivity` 의 notify 기본값·meta 형태를 바꾸면 **발주 알림 10종이 조용히 정지**. DB 기록은 정상이라 화면은 멀쩡, fire-and-forget 이라 에러도 없음 |
| `app/lib/b2b-auth.ts` | 6개 영역 18개 파일 + `middleware.ts` | 토큰 서명 방식 변경 시 타입·빌드 다 통과하고 **런타임에 전원 로그아웃** |
| `app/b2b/b2b.css` | **12개 화면 전부** (경로만 b2b) | 떨어진 위치에 같은 클래스명 → git 이 깨끗하게 합침 → 남의 화면 색이 말없이 바뀜 |
| `app/globals.css` (3~57줄 토큰) | 전 서비스 | 위와 동일 |
| `vercel.json` + `middleware.ts` | 크론·인증 | 아래 참조 |

- `b2b-activity.ts` 에 **자기 `log*()` 함수를 추가하는 건 괜찮다.** 공용 `recordActivity` 안쪽은 건드리지 말 것.
- 공용 파일을 꼭 고쳐야 하면 **다른 세션에 먼저 알린다.**

## 4. CSS 는 페이지 전용 접두사

`.b2b-*` 를 새로 정의하지 말고 `.ma-*`(메타광고), `.sales-*` 처럼 자기 페이지 전용 이름을 쓴다(기존 관행).
중복 검사:

```bash
grep -oE "^\.[a-zA-Z0-9_-]+ *\{" app/b2b/b2b.css | sed 's/ *{//' | sort | uniq -d
```

## 5. 크론은 늘리지 않는다

`vercel.json` 의 crons 는 Vercel Hobby **최대 2개**, 현재 1개 사용 중.
두 세션이 각각 1개씩 추가하면 **각자는 옳은데 합치면 3개** → **배포 전체 거부**. 로컬 빌드·타입체크로는 못 잡는다.
(이미 겪음: `543c59a "다이제스트 크론 복구: Hobby 하루1회로 배포 실패 해결"`)

크론 추가 시 `vercel.json` 과 `middleware.ts` 의 인증 예외를 **반드시 같이** 고칠 것. 한쪽만 하면 크론이 매일 401 을 맞고 조용히 죽는다.

## 6. dev 서버는 하나만

폴더 단위 잠금이라 **포트를 바꿔도 두 번째는 안 뜬다.**
살아있는 서버 하나가 **모든 세션의 편집을 자동 반영**하므로, 이미 떠 있으면 그 주소를 그대로 쓰면 된다. 죽이지 말 것.

## 프로젝트 규칙 (동시 세션과 무관)

- **마이그레이션은 절대 직접 적용하지 않는다.** 파일만 작성하고, 사용자가 Supabase SQL Editor 에서 적용한다. 적용 여부를 확인하고 안내할 것.
- 새 컬럼·테이블에 의존하는 코드는 **미적용 환경에서도 죽지 않게** 폴백을 넣는다 (기존 패턴: 에러 메시지에 컬럼명이 보이면 그 컬럼 빼고 재시도).
- **디자인 시스템 — UI(화면·컴포넌트·차트)를 만들거나 고치기 전에 `docs/design-system.md` 를 먼저 읽는다.**
  어떤 세션이 만들어도 같은 화면이 나오게 하는 단일 기준이다(결정표·함정 목록·색 지도가 거기 있다). 문서 없이도 지켜야 할 최소:
  - 토큰 단일 소스 = `app/globals.css` `:root`. **문서·주석·기존 코드에 적힌 hex 를 믿지 말고 :root 를 본다.** 인라인 hex/px 금지.
  - **없는 토큰을 지어내지 않는다.** `var(--sm-X, #폴백)` 은 X 가 :root 에 없으면 폴백이 항상 렌더된다(= 하드코딩인데 grep 에 안 걸림).
    쓰기 전 `grep -- "--sm-X:" app/globals.css` 로 존재 확인.
  - 있는 것을 재사용: 탭 `.sm-tab` · 배너 `.b2b-error`/`.sm-success`/`.sm-warn` · 배지 `.b2b-status-pill` ·
    차트는 `app/components/charts.tsx` 프리미티브(자체 차트 금지 — 표현이 부족하면 prop 을 추가).
    같은 의미는 lib 색 지도(`INV_TYPE_COLOR`·`VOC_STATUS_COLOR` 등)를 조회 — 화면에서 색을 새로 선언하지 않는다.
  - `:root` 토큰을 바꿨으면 `npm run sync-tokens` — public/ 정적 HTML 2개(iframe 도구)의 :root 가 따라온다.
    `npm run build` 는 이를 자동 실행하므로 배포 산출물은 항상 동기 상태다.
- **이모지 금지** (전문성).
- **번들(세트)은 자체 재고가 없다.** 현재고는 구성품에서 파생된다. 모든 입출고는 `app/lib/product-bundles.ts` 의 `expandBundleQty()` 로 구성품 전개해 기록할 것 — 경로마다 따로 구현하면 같은 판매가 경로에 따라 다르게 차감된다(실제 발생한 버그).
- **테스트가 없다.** 동작 변경을 잡아줄 장치가 `npx tsc --noEmit` + `npm run build` 뿐이므로, 의미가 바뀌는 수정은 직접 확인할 것.
- **사용 가이드(/guide) 동시 갱신 — 필수.** 화면·기능·메뉴를 추가/변경/삭제하는 커밋은 `app/lib/guide-data.ts` 를 **같은 커밋에서** 갱신한다(항목 추가/삭제, 단계·버튼명·메뉴명 일치, `GUIDE_UPDATED` 날짜). 가이드 문체: 부연설명 금지, 한 단계 = 한 동작, 화면의 실제 버튼명 그대로.
