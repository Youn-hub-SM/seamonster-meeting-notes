# 씨몬스터 B2B 관리툴 사용 설명서

도매(B2B) 발주의 등록부터 생산·발송·입금·정산까지 한 곳에서 관리하는 내부 도구입니다.

* 주소: **https://meeting-notes-beryl.vercel.app/b2b**
* PC·모바일 모두 지원합니다.

---

## 1. 시작하기

### 1.1 로그인

1. 주소로 접속하면 비밀번호 입력 화면이 나옵니다.
2. **본인 비밀번호**를 입력하세요. 비밀번호가 곧 본인 확인 수단입니다.
3. 로그인하면 화면 우측 상단에 **본인 이름**이 표시됩니다.

| 구분 | 설명 |
| --- | --- |
| 지인 / 예지 / 현석 | 각자 전달받은 본인 비밀번호 사용 |
| 관리자 | 관리자 비밀번호 사용 |

> 🔑 한 번 로그인하면 **30일간 유지**됩니다. 로그아웃은 우측 상단 [로그아웃].
>
> ⚠️ 비밀번호로 "누가 작업했는지"가 기록되므로 **다른 사람 비밀번호를 빌려 쓰지 마세요.**

### 1.2 메뉴 구성

상단 메뉴에서 각 화면으로 이동합니다.

| 메뉴 | 하는 일 |
| --- | --- |
| 대시보드 | 현황 요약 + 최근 변경 피드 |
| 발주 | 발주 등록·수정, 캘린더·주간·생산 집계, 발송요청 양식 |
| 업체 주소록 | 거래처 정보 관리 |
| 원가표 | 제품·옵션·원가·도매가 관리 |
| 이익률 | 제품별 이익률 조회 |
| 매출 집계 | 기간별 매출·마진 리포트, 엑셀 추출 |
| 입금 확인 | 미입금 발주 관리, 입금 내역 기록 |

---

## 2. 대시보드

로그인 직후 보이는 첫 화면입니다.

* **현황 카드** — 진행 중 발주, 입금 대기 등 주요 숫자를 한눈에 보여줍니다.
* **최근 변경 (우측 패널)** — 발주 등록, 상태 변경, 입금 기록이 실시간으로 쌓입니다.
  * 각 항목에 **작업자 이름과 시각**이 표시됩니다. (예: `지인 · 6월 10일 14:30`)
  * 20초마다 자동 새로고침되며, ↻ 버튼으로 즉시 갱신할 수 있습니다.
  * 항목을 클릭하면 해당 발주로 이동합니다.

## 3. 발주 관리

### 3.1 발주 목록

**발주** 메뉴의 기본 화면입니다.

* **검색**: 발주번호·업체명·메모·품목명으로 검색
* **필터**: 상태 / 업체 / 입금 / 세금계산서 / 품목 — 조합 사용 가능, [필터 초기화]로 해제
* **행 클릭** → 발주 상세(수정) 화면으로 이동
* **[복제]** → 같은 업체·상품 구성으로 새 발주 작성 (날짜·상태는 초기화)

#### 표에서 바로 바꾸기 (인라인 변경)

목록의 **생산 / 발송 / 입금 / 세금계산서** 칸은 드롭다운입니다. 발주를 열지 않고 바로 변경할 수 있고, 즉시 저장됩니다.

> 💡 발주 상태를 **발송완료**로 바꾸면 그 발주의 발송 일정들도 자동으로 발송완료 처리됩니다.

#### 지연 · 임박 뱃지

| 뱃지 | 의미 |
| --- | --- |
| 🔴 지연 | 발송예정일이 지났는데 아직 발송 안 됨 / 생산예정일이 지났는데 아직 생산 전 |
| 🟠 임박 | 발송예정일이 오늘 또는 내일 |

분할 발송 발주는 **아직 발송 안 된 다음 일정** 기준으로 판정하므로, 1차 발송을 마쳤다면 지연으로 뜨지 않습니다.

#### 일괄 작업

좌측 체크박스로 여러 발주를 선택하면 상단에 선택 바가 나타납니다.

* **상태 일괄 변경** — 선택한 발주들의 상태를 한 번에 변경
* **발송요청 양식 다운로드** — 3.4 참고

### 3.2 새 발주 등록

[+ 새 발주]를 누르고 위에서 아래로 채우면 됩니다.

#### ① 기본 정보

| 항목 | 설명 |
| --- | --- |
| 업체 * | 주소록에서 선택 (없으면 주소록에서 먼저 등록) |
| 발주일 * | 기본값 오늘. **매출 집계의 기준 날짜**입니다 |
| 생산예정일 | 생산 일정 관리·지연 판정에 사용 |
| 발송예정일 | 비워두면 발송 일정 중 가장 이른 날짜로 자동 입력 |
| 메모 | 포장 요청, 전달 사항 등 |

#### ② 상태

생산·발송·입금·세금계산서는 **서로 독립된 4개 상태**입니다. (예: 발송완료여도 입금은 입금전일 수 있음)

| 구분 | 값 (왼쪽 → 오른쪽 진행) |
| --- | --- |
| 생산 상태 | 생산대기 → 생산중 → 생산완료 |
| 발송 상태 | 발송대기 → 발송완료 (또는 취소) |
| 입금 상태 | 입금전 / 일부입금 / 입금완료 / 불필요 |
| 세금계산서 | 미발행 / 발행완료 / 불필요 |

#### ③ 배송 정보 (공통)

수령인 이름·연락처·주소·배송 메모·택배사를 입력합니다. **발송요청 양식에 그대로 들어가는 정보**입니다.

> 💡 비워두면 발송요청 양식에는 업체 주소록의 담당자·주소가 대신 사용됩니다.

#### ④ 발송 일정 (분할 발송)

한 발주를 여러 번 나눠 보낼 때 사용합니다. **한 번에 다 보내면 비워두세요.**

* [+ 발송 일정 추가] → 일정마다 입력:
  * **발송예정일** / **상태** (발송대기 → 발송완료, 취소)
  * **보낼 수량 (상품별)** — 이 차수에 보낼 수량만 입력
  * **운송장 번호** (선택)
* 캘린더·주간 뷰에 일정별로 따로 표시되고, 발송요청 양식도 일정 단위로 뽑을 수 있습니다.

> ⚠️ 분할 수량 합계가 발주 수량과 다르면 노란 경고가 표시됩니다. (예: "발주 10개인데 발송 일정에 7개 배분됨") 저장은 가능하니 의도한 게 맞는지만 확인하세요.

#### ⑤ 발주 상품

* **제품 선택** 드롭다운에서 고르면 품목명·옵션·단가(도매가)·과세 구분이 자동 입력됩니다.
  * 표시 형식: `삼치순살(5kg x 2) 100g` = 품목명 + 옵션
* 목록에 없는 상품은 **직접 입력**으로 품목명·옵션·단가를 손으로 채울 수 있습니다.
* 수량·단가는 자동 입력 후에도 수정 가능합니다.

#### ⑥ 합계 · 이익률

* **합계** — 과세분 / 면세분 / 부가세(과세분×10%) / 총합계가 자동 계산됩니다.
* **이익률 (배송 박스 기준)** — 이 발주의 예상 손익을 보여줍니다.

```
이익 = 매출 − 제품원가 − 배송비
배송비 = 배송 박스 수 × (아이스박스 + 운반비 + 보냉비)
```

| 입력 | 설명 |
| --- | --- |
| 배송 박스 수 | 실제 나갈 박스 수 입력. 아래에 총 부피와 권장 박스 수(박스당 최대 20kg)가 안내됩니다 |
| 계절 (자동) | 발송예정일 기준 — 동절기(12·1·2월) / 하절기(3·4·5·6·10·11월) / 극하절기(7·8·9월). 보냉비가 달라집니다 |

> 💡 과세 상품은 매출을 공급가(÷1.1)로 환산해 계산합니다. 박스 수를 바꾸면 즉시 다시 계산됩니다.

#### 저장

* [등록]을 누르면 발주번호가 자동 발번됩니다. (형식: `20260610-001`)
* "발주번호 발번이 겹쳤습니다"가 뜨면 **한 번 더 저장**을 누르면 됩니다. (동시 등록 시 드물게 발생)

### 3.3 발주 수정 · 삭제

* 목록에서 발주를 클릭 → 같은 화면에서 수정 후 [수정]
* [삭제]는 발주 상품·발송 일정까지 함께 삭제됩니다. **복구 불가**이니 신중히.

> ⚠️ **같은 발주를 두 명이 동시에 열어 수정하면 나중에 저장한 내용이 먼저 저장한 내용을 덮어씁니다.** 발주별 담당자를 정해 두는 것을 권장합니다.

### 3.4 발송요청 양식 다운로드

물류처에 보내는 발송요청 엑셀을 자동 생성합니다.

1. 발주 목록에서 보낼 발주들을 체크
2. [발송요청 양식 다운로드] 클릭
3. 분할 발송이 있는 발주가 포함되면 **"어떤 발송을 뽑을까요?"** 선택 창이 뜹니다.
   * 발송 일정별로 날짜·상태·상품 요약이 표시되고, 출력할 일정만 체크
   * 취소된 일정은 기본 해제되어 있습니다
4. 엑셀에는 상품명·옵션·수량·수령인·연락처·주소·배송메모가 채워집니다.

> 💡 분할 발송이 없으면 선택 창 없이 바로 다운로드됩니다.

### 3.5 캘린더 뷰

발주 목록의 **[캘린더]** 탭.

* PC는 월간, 모바일은 3주 단위로 표시
* 상단 토글로 **발주일 / 생산일 / 발송일** 표시 여부 선택
* 분할 발송은 **발송 일정별로 각 날짜에 따로** 표시되고, 일정 상태(발송대기/발송중/발송완료) 뱃지가 붙습니다
* 항목 클릭 → 해당 발주로 이동

### 3.6 주간 뷰 (발송일)

발송일 기준으로 주 단위 그룹을 보여줍니다. 이번 주가 강조되며, 분할 발송은 일정별로 행이 나뉩니다. 주별 발송 건수와 금액 합계를 확인할 수 있습니다.

### 3.7 생산 집계

기간을 정하면 **품목+옵션별 생산해야 할 수량**을 합산해 보여줍니다. 생산 계획을 잡을 때 사용하세요.

## 4. 업체 주소록

거래처 정보를 관리합니다.

* [+ 업체 추가] → 업체명(필수), 사업자번호, 대표자, 담당자, 연락처, 이메일, 주소, 결제 조건, 메모
* 업체를 클릭하면 상세 화면에서 **그 업체의 발주 이력**을 볼 수 있습니다.
* **담당자·연락처·주소**는 발주의 배송 정보가 비어 있을 때 발송요청 양식에 대신 사용됩니다.

> ⚠️ 발주가 있는 업체는 삭제할 수 없습니다. (이력 보호)

---

## 5. 원가표

제품·옵션·원가·도매가를 관리합니다. **발주 등록과 이익률 계산의 기준 데이터**입니다.

### 5.1 제품 구조

| 항목 | 예시 | 설명 |
| --- | --- | --- |
| 품목명 | 삼치순살(5kg x 2) | 용량 포함 제품 이름 |
| 옵션 | 100g / 200g | 같은 제품의 세부 구분 |
| SKU | BULK-DSC-100 | 내부 상품코드 (100=100g, 200=200g) |
| 과세 유형 | 면세 / 과세 | 면세면 부가세 계산에서 제외 |

### 5.2 가격 입력

| 항목 | 설명 |
| --- | --- |
| 기본 판매가 (도매가) | **발주서에서 단가로 자동 입력되는 값** |
| 제품원가 | 제조 원가 |
| 내포장지 / 라벨 / 외포장지 | 포장재 단가 — 제품원가와 합쳐 "제품 단위 원가"가 됩니다 |
| 제품부피 (kg) | 발주 이익률의 박스·배송비 계산 기준 |

* 원가를 수정하면 **원가 이력**이 자동 기록됩니다. ([원가 이력] 버튼으로 조회)
* 메모에는 소비자가, 100kg 이상 대량가, 쿠팡가가 참고로 적혀 있습니다.

### 5.3 미사용 처리

판매를 멈춘 제품은 삭제 대신 **"사용 중" 체크를 해제**하세요. 발주 등록 드롭다운과 이익률 목록에서 빠지고, 과거 발주 이력은 유지됩니다. (발주에 들어간 제품은 삭제도 막혀 있습니다)

## 6. 이익률

제품별 수익성을 한눈에 봅니다.

```
이익률 = (매출 − 제품 단위 원가) ÷ 매출
매출 = 도매가 (과세 상품은 공급가 ÷1.1)
제품 단위 원가 = 제품원가 + 포장재(내/라벨/외)
```

* 상단 KPI: 대상 제품 수 / 평균 이익률 / 흑자·적자 수
* [원가] 버튼으로 행을 펼치면 원가 구성과 손익 상세를 볼 수 있습니다.
* 배송비는 발주마다 박스 수가 달라 여기서는 빼고 계산합니다. **배송비 포함 손익은 각 발주 상세의 "이익률" 영역**에서 확인하세요.

---

## 7. 매출 집계

기간별 매출 리포트입니다. **발주일 기준, 취소 제외 전체 발주**를 집계합니다.

* 기간 프리셋: 이번 달 / 지난달 / 올해 / 직접 지정

| KPI | 의미 |
| --- | --- |
| 총 매출 | 기간 내 발주 합계 (과세분·면세분 구분 표시) |
| 발주 건수 | 기간 내 발주 수 · 건당 평균 |
| 예상 마진 | 각 발주의 이익(공급가 매출 − 발주시점 원가 − 박스 배송비)을 **합산**한 값 |
| 미발송 잔고 | 아직 발송 안 된 발주 금액 (기간 무관) |

* **월별 매출 추세** / **업체별 매출·마진** / **제품별 매출** (품목+옵션 단위, 수량·원가·마진율)
* [엑셀 다운로드] — 발주 상품을 한 줄씩 펼친 엑셀 파일

> 💡 원가는 "발주를 등록한 시점"의 원가가 박제되어 쓰입니다. 이후 원가표를 바꿔도 과거 발주 마진은 변하지 않습니다.

## 8. 입금 확인

돈 들어올 발주를 관리합니다.

1. 미입금·부분입금 발주가 목록에 표시됩니다 (청구액·입금액·잔액).
2. 발주를 클릭해 **입금 내역 추가** — 금액, 입금일, 방법, 적요
3. 잔액이 0이 되면 발주의 입금 상태를 **입금완료**로 바꿔 마감합니다.

입금 기록도 활동 피드에 작업자 이름과 함께 남습니다.

## 9. 자주 묻는 질문 (FAQ)

**Q. 비밀번호가 틀렸다고 나와요.**
본인 비밀번호인지 확인하세요. 분실 시 관리자에게 문의하면 재설정해 드립니다.

**Q. 저장할 때 "발주번호 발번이 겹쳤습니다"가 떴어요.**
두 명이 동시에 발주를 등록할 때 드물게 발생합니다. **한 번 더 저장**하면 됩니다.

**Q. 발송요청 양식에 주소가 "(주소 미입력)"으로 나와요.**
그 발주의 **배송 정보**가 비어 있고 업체 주소록에도 주소가 없는 경우입니다. 둘 중 한 곳에 주소를 채우고 다시 다운로드하세요.

**Q. 분할 발송했는데 캘린더에 계속 "발송대기"로 보여요.**
발송 일정의 상태를 발송완료로 바꾸지 않은 경우입니다. 발주를 열어 해당 일정 상태를 바꾸거나, 전부 나갔다면 발주 상태를 **발송완료**로 바꾸세요 (일정도 자동 완료 처리됩니다).

**Q. 이익률 페이지와 발주의 이익률 숫자가 달라요.**
이익률 페이지는 **배송비 제외·현재 원가** 기준, 발주 상세는 **배송비 포함·발주 시점 원가** 기준이라 다를 수 있습니다. 발주 손익은 발주 상세가 정확합니다.

**Q. 노란 경고("발주 10개인데 7개 배분됨")는 뭔가요?**
발송 일정에 나눠 넣은 수량 합이 발주 수량과 다르다는 알림입니다. 아직 일정을 다 안 잡았다면 정상이고, 다 잡았는데 뜬다면 수량을 다시 확인하세요.

**Q. 발주를 잘못 삭제했어요.**
삭제는 복구되지 않습니다. 발주 상품·발송 일정·송장 정보가 함께 사라지니, 삭제 대신 상태를 **취소**로 바꾸는 것을 권장합니다.

**Q. 두 명이 같은 발주를 고치면 어떻게 되나요?**
나중에 저장한 사람의 내용으로 덮어써집니다. 같은 발주는 한 사람이 맡아서 수정하세요.

## 10. 작업 흐름과 역할 분담 (R&R)

발주 한 건은 **접수 → 생산·발송 → 정산** 순서로 흐릅니다. 단계마다 담당이 정해져 있으니, **자기 차례의 상태만** 챙기면 됩니다.

### 10.1 누가 무엇을 맡나

| 담당 | 한마디로 | 맡는 일 | 주로 쓰는 메뉴 |
| --- | --- | --- | --- |
| **예지** | 받아서 등록 | 업체 응대 → 발주 접수·등록 (상품·수량·생산예정일·배송 정보·발송 일정 입력) | 발주 [+ 새 발주], 업체 주소록 |
| **지인** | 만들어서 내보내기 | 생산 상태 체크(생산중·생산완료) → 발송요청 양식·물류 전달·운송장 입력 | 발주 목록(상태 변경), 발송요청 양식, 생산 보드 |
| **현석 / 관리자** | 돈 받고 정산 | 입금 확인 · 세금계산서 발행 · 월말 매출 집계 | 입금 확인, 매출 집계 |

> 💡 한 줄 요약: **예지 = 입력**, **지인 = 생산·발송**, **현석/관리자 = 정산**.

### 10.2 단계별 흐름 (담당 · 바뀌는 상태)

| # | 단계 | 담당 | 하는 일 | 바뀌는 상태 |
| --- | --- | --- | --- | --- |
| 1 | 발주 접수·등록 | **예지** | 업체·상품·수량 입력(도매가 자동), 생산예정일·배송 정보·발송 일정 입력 후 [등록] | 생산 `생산대기` · 발송 `발송대기` (초기) |
| 2 | 생산 지시 | **지인** | 생산 들어가면 생산 상태를 변경 | 생산 → `생산중` |
| 3 | 생산 완료 | **지인** | 생산이 끝나면 변경 | 생산 → `생산완료` |
| 4 | 발송 | **지인** | 발송요청 양식 다운로드 → 물류 전달 → 운송장 입력 후 변경 | 발송 → `발송완료` |
| 5 | 입금 | **현석/관리자** | 입금 확인에서 입금 내역 기록, 잔액 0이면 변경 | 입금 → `입금완료` |
| 6 | 세금계산서 | **현석/관리자** | 발행 후 변경 | 세금계산서 → `발행완료` |
| 7 | 월말 정산 | **현석/관리자** | 매출 집계에서 실적·마진 확인, 엑셀 추출 | — |

### 10.3 인수인계 — 헷갈리기 쉬운 경계

* **예지 → 지인**: 예지가 발주를 저장하면(`생산대기`) 지인 차례입니다. 지인은 발주 목록에서 `생산대기`·`생산중` 발주를 챙기고, **생산 보드**(생산관리 툴)에서 밀린 생산을 한눈에 봅니다.
* **지인 → 현석/관리자**: 지인이 **발송완료**로 바꾸면 매출 시트에 자동 기록되고, 그때부터 입금·세금계산서는 현석/관리자가 이어받습니다.
* **상태 4가지(생산·발송·입금·세금계산서)는 서로 독립**입니다. 예를 들어 *발송완료여도 입금은 `입금전`* 일 수 있어요. 각자 자기 담당 상태만 보고 처리하면 서로 충돌하지 않습니다.

> ⚠️ 같은 발주를 두 명이 동시에 열어 수정하면 나중에 저장한 사람이 덮어씁니다. 위 담당 구분대로 **자기 단계에만 그 발주를 여는 것**을 원칙으로 하세요.
